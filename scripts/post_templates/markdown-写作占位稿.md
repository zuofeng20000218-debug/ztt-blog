这篇文章用来测试博客正文排版。以后可以替换成真正的 Markdown 写作说明。

## 标题

正文里会出现二级标题、三级标题和普通段落。标题需要清楚，但不要太花。

### 小节

小节用于把复杂内容拆开，让读者更容易扫读。

## 列表

- 第一条记录
- 第二条记录
- 第三条记录

## 引用

> 写博客最重要的不是一开始就完美，而是让内容能被持续补上。

## 代码块

```ts
const blog = {
  framework: 'Astro',
  hosting: 'Cloudflare Pages',
  status: 'draft',
};
```

如果这些元素在页面里看起来舒服，后面写长文就会轻松很多。
